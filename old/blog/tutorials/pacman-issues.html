This is a blog post where I track pacman issues and fixes.

05/06-11-2022

I was getting an error with pacman:

error while loading shared libraries: libssl.so.1.1: cannot open shared object file: No such file or directory

This happened when I tried to install my VPN daemon using the AUR, and that tried to update a single package instead of all system packages, and thus I had some issues with the libssl package. This broke pacman, sudo, systemd and MUCH more.

{Solution}

The only thing I needed to do is to reinstall openssl-1.1.0. So what I did is:

  1. Boot into an Arch GNU/Linux live environment.
  2. Mount the system partition (not boot, just /) using:

mount /dev/nvme0n1p2 /mnt

OR

mount /dev/sda2 /mnt

  1. Connect to the internet (iwd is easier than you might think)
  2. Pacstrap the openssl-1.1.0 package:

pacstrap -i openssl-1.1.0 /mnt

  1. Make sure it works; chroot into the drive and do something like pacman --help.

arch-chroot /mnt

pacman --help

  1. Umount your normal boot drive and reboot the system:

umount /dev/nvme0n1p2

reboot now
