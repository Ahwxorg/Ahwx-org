Debian install guide for server usage

Debian is a minimalistic GNU/Linux distribution, built around the APT package manager.

Installing on a VPS

Most VPS providers give you the option to install Debian, just select that option. If you do, you’re already done!

Installing on a PC/server

You can find the installer for Debian on their website. Usually you want AMD64 (unless you run ARM) and netinstall is fine if your server has internet access (which it should).

Burning the ISO to a USB drive

You can use any tool to burn the ISO to any USB drive. I would personally recommend Rufus if you’re running Windows. For MacOS, I have no idea. For GNU/Linux, you can use dd using this guide.

Setting up language based things

You’re probably greeted with the following screens if you’re booted correctly. Please follow along.

Select “Graphical Debian Installer” and hit enter.

Now, select the imformations that works for you.

Once you’ve finished that step, we’re going to:

Configure the network

A hostname is a name which your computer has and shows in the network, to help identify it. You can make anything up. I personally use proxbox.pve. This is because my server box runs Proxmox and PVE is short for Proxmox Virtual Environment.

Create accounts

You’ll want to be able to log into your new server, and so you’ll have to make user accounts. One for root, and one for your actual user. As this is fairly straight forward, no screenshots here.

Partitioning the disks

Since your server needs to store it’s data somewhere, we need to configure the disks. You can do this at any way, but for simplicity sake, I’d recommend starting with “Guided - use entire disk”.

Warning: this will erase any data on any disk you select!

After that’s done, the installation should start.

Final touches

Make sure to select “SSH server” and “standard system utilities”, deselect “web server” if it’s checked (even if you plan on hosting a web server) and deselect any DE
